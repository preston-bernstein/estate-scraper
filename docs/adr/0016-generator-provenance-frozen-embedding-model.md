# 0016 — Generator provenance and a frozen embedding model

Every Image embedding, Finding description, and Item identification is the output of a specific embedding model, VLM, and prompt — and over a corpus that accumulates for a year and trains a ranker, leaving that implicit is silent corruption. Three failures follow from an unstamped corpus: embeddings from two different models share a `BLOB` column despite living in incompatible geometries (cosine distance between a CLIP and a SigLIP vector is noise); `desirability='high'` means different things before and after a prompt tweak, so the ranker trains across the seam; and without a version stamp there is no way to re-score what an old prompt missed or to A/B two models on the same Images. The decision is to **stamp the generator on every generated row** — `embed_model` + `embed_dim` on Image, `vlm_model` + `prompt_version` on Finding and Item — using inline columns rather than a normalized `model_runs` table, since model swaps are rare events, not a high-cardinality dimension.

The embedding model specifically is **frozen to one choice**, because embeddings are the single tier that [ADR 0013](0013-persist-embeddings-thumbnails-at-scan-time.md) makes irreplaceable — the source listing is gone, so a vector in the wrong space cannot be recomputed from the original. A column mixing two embedding spaces is therefore not a degradation but a total loss of the year's most valuable data. Changing the embedding model is consequently defined as an explicit **corpus migration**: re-embed every Image from its persisted thumbnail and rewrite the column wholesale, never append vectors from a new model alongside the old. This retroactively makes the thumbnail in ADR 0013 load-bearing beyond the UI — it is the only artifact that survives the source listing and lets a model change become a re-embed job instead of data loss. The VLM and prompt are *not* frozen (they are expected to be tuned), which is precisely why their version stamp is mandatory: it marks the seams so training and backfill can respect them.
