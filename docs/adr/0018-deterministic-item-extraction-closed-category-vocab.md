# 0018 — Deterministic finding_items extraction: one item per VLM line, closed category vocab, lexicon as sole maker authority

[ADR 0014](0014-image-finding-item-three-level-model.md) defined the `finding_items` tier — the normalized, queryable identification (maker, category, era, desirability) split out of a Finding's free text — but not how the rows get produced. The options were a second LLM pass to structure each Finding, or a deterministic parser over the text the VLM already emits. The decision is **deterministic extraction, no extra inference**: the vision prompt already returns one notable item per line (`lib/vision.ts`), so each non-empty line of `findings.description` becomes one `finding_items` row. This is free (no GPU/cost per item), idempotent (the same Finding always yields the same rows, so a lexicon-growth re-mine is safe to re-run), and trivially testable. An LLM re-extraction pass can be layered on later for the lines this misses; it is not needed to make the tier useful now.

**Category is a closed vocabulary of 15** (`seating, tables, case_goods, beds, lighting, clocks, art, ceramics_glass, silver, jewelry_watches, electronics, games_toys, instruments, kitsch, other`), mirroring the taste areas in the vision prompt. It is a facet, not a free-text field, so unmatched lines fall to `other` rather than minting new categories. Classification is ordered keyword matching, most-specific-first, so `kitsch` and named electronics win over the broad furniture buckets. Measured on the 4,419-finding backfill, ~45% of items land in `other` — accepted, because estate sales are full of clothing, kitchenware, and bric-a-brac outside the buyer's three taste areas, and inflating the categories to absorb that noise would make the facet less useful, not more.

**The lexicon (`lib/lexicon.ts`) is the sole authority for `maker`** (ADR 0014): `maker` only ever holds a canonical name resolved from the lexicon, with `maker_raw` keeping the matched raw spelling so rows can be re-mined as the lexicon grows. Alias matching is bounded by non-alphanumerics so `lane` doesn't fire inside `planer`; the longest alias wins so `atari 2600` beats `atari`. **Desirability is distinct from identification confidence** (CONTEXT.md): a lexicon maker hit or a premium era signal lifts an item to `high`, otherwise items the VLM bothered to list sit at `med`; human Outcomes later override to gold (ADR 0015), never market data. Every row is stamped with `vlm_model` + `prompt_version` (ADR 0016) so a prompt change marks a seam in the corpus. The same extraction runs inline for new findings during a scan and in a `backfill:items` job for the pre-existing corpus and for lexicon-growth re-mines.
